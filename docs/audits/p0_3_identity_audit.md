- `README.md`: contains "Team A / Team B configuration files" in the included pieces list. Update to fictional launch identities plus a safety note that identities inherit behavior from archetypes and graph concepts.
- `ui/showcase_manifest.json`: launch gallery labels are currently handle-style names. Update visible labels to fictional identity display names and preserve the original strings in `technical_label` fields.

- `scripts/run_best_of_n.py`, `run_match_matrix.py`, `run_matchup.py`, `run_comparison_report.py`, and `build_replay_index.py`: keep existing Team A/B language where it describes technical benchmark output, not launch UI.
- `agents/static_*`, `agents/adaptive_*`, generated replay/report JSON under `data/` and `ui/showcase_replays/`, and historical audit/plan docs: keep unless regenerated by their owning phase. These are technical artifacts and golden/test fixtures.
- Tests referencing `team_a_static_baseline` or `team_b_adaptive_counter`: keep because they pin benchmark behavior.

## Decision

Create a fictional identity layer that maps to existing Agent Garage archetypes and graph concepts. Do not rename or delete benchmark files. Launch surfaces should show identity names by default; technical labels remain available for debug and practitioner contexts.
